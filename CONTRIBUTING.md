# Contributing to AutoScan Research

Thank you for your interest in the AutoScan research project! This document provides guidelines for contributing to the research paper repository.

## Types of Contributions

We welcome various forms of contributions:

### 1. Research Feedback
- Suggestions for methodology improvements
- Questions about experimental design
- Ideas for additional experiments or metrics
- Discussion of related work

### 2. Documentation
- Corrections to technical descriptions
- Clarifications in methodology
- Additional examples or use cases
- Improved explanations

### 3. Code Examples (Future)
- If/when code is released, contributions to implementation
- Bug fixes and optimizations
- Additional features and modules

### 4. Dataset and Examples
- High-quality example images (with proper licensing)
- Additional test cases
- Edge cases and challenging scenarios

## How to Contribute

### Reporting Issues

If you find errors or have suggestions:

1. **Check existing issues** to avoid duplicates
2. **Open a new issue** with:
   - Clear description of the issue/suggestion
   - Relevant section or page reference
   - If reporting an error, include the correct information
   - If suggesting improvements, explain the rationale

### Suggesting Improvements

For research methodology or experimental suggestions:

1. **Open an issue** describing your suggestion
2. Include:
   - Background and motivation
   - Proposed approach
   - Expected benefits
   - Relevant citations or prior work

### Contributing Documentation

For documentation improvements:

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b improve-methodology-docs`
3. **Make your changes**
4. **Commit with clear messages**: `git commit -m "Clarify text detection methodology"`
5. **Push to your fork**: `git push origin improve-methodology-docs`
6. **Open a Pull Request** with:
   - Description of changes
   - Motivation for the changes
   - Any related issues

## Contribution Guidelines

### Documentation Style

- Use clear, concise language
- Follow existing formatting conventions
- Include citations where appropriate
- Maintain consistent terminology
- Use proper academic writing style

### Technical Accuracy

- Ensure all technical details are accurate
- Verify any claims with evidence or citations
- Use appropriate units and measurements
- Include proper mathematical notation when needed

### Code Style (if applicable)

If code contributions are accepted in the future:
- Follow PEP 8 for Python code
- Include docstrings for functions and classes
- Add comments for complex logic
- Include unit tests where appropriate

## Research Ethics

When contributing:

- Respect copyright and intellectual property
- Properly attribute ideas and prior work
- Use only appropriately licensed images/data
- Follow research ethics guidelines
- Maintain scientific integrity

## Copyright and Licensing

By contributing, you agree that:

- Your contributions will be licensed under the MIT License
- You have the right to submit the contributions
- Your contributions are your original work or properly attributed

## Questions and Discussion

For questions about contributing:

1. Check existing issues and discussions
2. Open a new issue with the "question" label
3. Be respectful and constructive in discussions

## Recognition

Contributors will be acknowledged in:

- The repository's contributor list (automatically generated by GitHub)
- Acknowledgments section of future paper versions (for significant contributions)
- Relevant documentation sections

## Contact

For private inquiries about contributions:
- Open an issue (preferred for transparency)
- Contact repository maintainers via GitHub
